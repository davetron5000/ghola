The g-palette-color component represents one color with many shades in your palette. For example the chosen purple and then darker and lighter shades. If it contains a g-color-name, it can expose the name and if the user has overridden the name. If it contains one or more g-color-swatch elements, it can return an ordered list of the hexcodes.

When you combine this with the g-color-name, g-color-swatch elements with derivations, you can createa color palette.

Attributes

The g-palette-color recognizes three attributes:

Attribute Values Notes
primary Set or omitted

If set, this color is considered the primary one in the palette and cannot be removed or unlinked.

linked-to The id of another g-palette-color.

If set, this can be unlinked.

linked-to-primary Set or omitted

If set, the component will find a g-palette-color with primary set in the children if this component's parent.

Events

The g-palette-color also recognizes three buttons that can trigger special events:

Action Event Name Markup Needed Notes
Preview preview One or more elements with data-preview

No special behavior other than connecting the tagged elements with the higher-level event.

Unlink unlink One or more elements with data-unlink

Elements will trigger the higher-level event, but also the linked-to attribute is removed. You can stop this via preventDefault

Remove remove One or more elements with data-remove

Elements will trigger the higher-level event, but this also removes the entire g-palette-color from the DOM. You can stop this via preventDefault

API

The g-palette-color can provide programmatic access to what it's wrapping.

Method Return Value Notes
.colorName Name of the color, as a String

If set, this color is considered the primary one in the palette and cannot be removed or unlinked.

.colorNameUserOverride true or false

true if the user has overridden the color, false if the default system-provided value is being used.

.colorScale Array of hex codes as strings

Values are provided in the order of the g-color-swatch elements. Empty if there aren't any.

Edge Cases

debug can be set to cause console logs to generate when an edge case or obvious mis-use occurs.

  • linked-to is set, but there is no element with that id.

  • linked-to-primary is set, but there is no primary with that id.

  • linked-to-primary and linked-to are set, which results in undefined behavior.
